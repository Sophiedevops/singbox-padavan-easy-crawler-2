## 🇷🇺 Русский

**Автоматический поиск, тест и прямое управление живым процессом Sing-Box на слабом MIPS-роутере**

**( протестировано на MiWiFi 3 | Xiaomi Mi-router 3 )**
> **Единственный скрипт**, который переварит базу из **240 000+ прокси-нод**
> на роутере с **32 МБ свободной ОЗУ** — без зависания, без OOM, без перегрева.

> **🆕 Что изменилось в актуальной версии `update.sh`:**
> - Скрипт теперь сам **управляет live-процессом** sing-box через PID-файл (запуск/остановка/горячий рестарт).
> - **Provider Auto-Pilot** — автоматический выбор рабочего CDN для спид-теста и пинг-эндпоинта (обход блокировок Cloudflare/CacheFly).
> - **Diamond Extractor (Алмазный экстрактор)** — VIP-страны (NL, DE, US и т.д.) гарантированно вытаскиваются из 240k+ баз регулярными выражениями без случайного отбрасывания при защите ОЗУ.
> - **Динамический RAM-контроль** — турбо-режим в `tmpfs` включается только при >150 МБ ОЗУ. Иначе используется безопасная папка на флешке (`sb_upd_tmp`).
> - Base64-декодер переписан на **чистом Lua**, а фильтры `jq` генерируются динамически для обхода багов урезанного BusyBox (`ash`).
> - **Anti-Zombie Защита** — перехватчик `Ctrl+C` и глубокое сканирование `/proc/*/cmdline` гарантированно убивают зависшие тестеры, не трогая главный сервер.
> - Повышены пороги: `PERFECT_SPEED_KBPS` 800→**900**, `MIN_FAST_CHECK_SPEED_KBPS` 500→**600**.

### 🔄 Как устроен пайплайн (порядок работы)

1. **Smart Fast-Check.** Если `conf2_final.json` существует, скрипт проверяет ноды текущего пула: пинг через API, затем реальный спидтест. Если стабильных нод ≥70% от `WANTED` — весь тяжёлый пайплайн пропускается, поднимается live-процесс, скрипт завершает работу.
2. **Скачивание и авто-декодирование.** Опрашиваются 8 агрегаторов. Если текст не похож на прокси-ссылки, он прогоняется через встроенный Lua-декодер Base64; мусорные байты вычищаются автоматически.
3. **Алмазный экстрактор + защита ОЗУ.** Текстовые ссылки фильтруются `grep` по тегам целевых стран из `FILTER_COUNTRIES`. Формируются 4 безопасные выборки (VIP-SS, VIP-другие, обычные-SS, обычные-другие). Только после этого они конвертируются в JSON.
4. **Глобальная матрица сортировки.** Ноды раскладываются по приоритету согласно `ENCRYPTION_PRIORITY` и `SORT_PRIORITY`, плюс отдельный fallback-блок для стран вне списка.
5. **Цикл сканирования батчами по 3 ноды.** Для каждой партии поднимается тестовый sing-box; пинг проверяется через API, прошедшие — тестируются на скорость. Найденные ноды (≥ `PERFECT_SPEED_KBPS`) копятся в результат.
6. **Сборка и горячее переключение.** Из лучших нод собирается `Best-Auto`, мёрджится в шаблон `conf3_final.json` и пишется в `conf2_final.json`. Live-процесс перезапускается без разрыва интернет-соединения (около 1 сек).

### 📦 Состав проекта

| Файл | Назначение |
| --- | --- |
| `./update.sh` | 🔧 **Главный движок** — пайплайн + встроенный supervisor |
| `./gen_links.sh` | 🔗 **Генератор ссылок** — готовые URI для локальной сети |
| `converter.lua` | ⚙️ **Парсер URI** — конвертирует `ss://` и др. в формат Sing-Box |
| `utils.lua` | 🛠️ Вспомогательные функции для Lua |
| `conf3_final.json` | 📋 **Шаблон (база)** — используется как цель мёрджа, не редактируется напрямую |
| `conf2_final.json` | ⚡ **Активный конфиг** — перезаписывается скриптом; исполняется боевым процессом |

> 💡 `update.sh` сам пересоздаёт live-процесс sing-box по PID-файлу (`/var/run/sb_update_main.pid`). Автозапуск при перезагрузке роутера обеспечивает ваш init-скрипт/cron.
